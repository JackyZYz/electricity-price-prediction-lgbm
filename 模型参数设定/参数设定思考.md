模型训练参数的思考可以从三个层面展开：**任务特性 → 参数作用 → 调参策略**。下面是针对本项目的系统思路：

---

## 一、先理解任务特性

| 特性     | 本项目情况                             | 对参数的影响                                        |
| -------- | -------------------------------------- | --------------------------------------------------- |
| 数据量   | 约 176 天 × 96 点 ≈ 16,900 条          | 样本量中等，树模型可以胜任，但不用太深              |
| 目标分布 | 电价均值 310，标准差 57.8，尖峰达 1251 | 对异常值敏感，需控制过拟合；可考虑 Huber/分位数损失 |
| 时间结构 | 强日周期、周周期、季节趋势             | 日期特征很重要，时序分割必须防泄露                  |
| 特征维度 | 约 28 维，含构造/滞后/日期             | feature_fraction 不宜过高，避免部分特征主导         |
| 预测目标 | 日前预测（提前 1 天）                  | 滞后特征可用，但只能用 T-1 及之前                   |

---

## 二、LightGBM 核心参数的思考

### 1. 学习率 `learning_rate` 与迭代次数 `num_boost_round`
- **关系**：学习率越小，达到同样效果需要更多树。
- **本项目**：当前 `0.05` 配合 `early_stopping=50` 是合理起点。
- **调参方向**：
  - 若 early stopping 时迭代次数还远未到上限（如 `<1000`），可保持 `0.05`。
  - 若早停时接近 `5000`，说明学习率偏大或模型容量不够，可降到 `0.01~0.03`。
  - 原型中 `num_boost_round=2000`、`lr=0.05` 能收敛，说明当前量级合适。

### 2. 树复杂度控制：`num_leaves`、`max_depth`
- **作用**：控制单棵树的拟合能力。
- **本项目**：
  - `num_leaves=63`、`max_depth=12` 属于中等偏强。
  - 由于有强周期特征（小时、是否峰段），模型不需要极深的树就能学到。
  - 若验证集损失下降但方向准确率差，可能是过拟合，应降低 `num_leaves` 到 `31~50`，`max_depth` 到 `8~10`。

### 3. 叶子最小样本 `min_data_in_leaf`
- **作用**：防止叶子节点样本过少导致过拟合。
- **本项目**：`20` 是通用值。考虑到 15 分钟数据噪声较大，可尝试 `50~100`，提升泛化。
- 但太大可能欠拟合，需结合 CV 看。

### 4. 特征/样本采样：`feature_fraction`、`bagging_fraction`
- **作用**：随机选择特征和样本构建每棵树，降低过拟合。
- **本项目**：`0.8` 是稳健起点。若特征重要性很集中（如 `hour`、`price_lag_1d` 占 80%），可降到 `0.6~0.7` 强制模型学习更多特征组合。

### 5. 正则化：`lambda_l1`、`lambda_l2`
- **作用**：控制叶子权重和分裂增益的惩罚。
- **本项目**：`0.01` 属于轻正则。电价尖峰多、噪声大，可尝试 `lambda_l1=0.1`、`lambda_l2=0.1` 增强稳定性。

### 6. 损失函数
- 当前默认 `regression` + `rmse`。
- 由于电价尖峰极端，可考虑：
  - `objective: huber`：对异常值更稳健。
  - `objective: quantile` + `alpha: 0.5`：中位数回归，降低尖峰影响。
  - `fair` / `mape` 等自定义损失。

## 三、小波分解参数

| 参数              | 当前值 | 思考                                               |
| ----------------- | ------ | -------------------------------------------------- |
| `wavelet`         | `db4`  | 常用选择，适合电价这种非平稳信号                   |
| `decompose_level` | `2`    | 2 层可分离日周期和更细波动；可尝试 1 层或 3 层对比 |
| `window_size`     | `30`   | 用于滚动分解的历史窗口；与 `test_days` 独立        |

**关键问题**：小波分解后，低频/高频分量的长度会缩短。需要保证：
- 训练目标长度与特征矩阵对齐。
- 当前 `train_pipeline.py` 中直接用 `y_train` 做分解，但 X_train 未做相应截取，存在 bug，需要修复。

---

## 四、时序分割与验证策略

| 参数        | 当前值 | 思考                                 |
| ----------- | ------ | ------------------------------------ |
| `test_days` | `30`   | 约 2880 条测试样本，合理             |
| `cv_folds`  | `5`    | 时序 CV 应该用滑窗，避免未来信息泄露 |

**建议**：
- 原型阶段用固定 `train/test` 分割快速验证。
- 调参阶段使用时序交叉验证（如 `TimeSeriesSplit` 或滑窗 CV）。
- 不要随机 CV，否则会泄露未来信息。

## 五、超参数调参策略

可以用 Optuna 自动搜索，但搜索空间要有业务约束：

```yaml
lgbm_params_search:
  learning_rate: [0.01, 0.05, 0.1]
  num_leaves: [31, 63, 127]
  max_depth: [6, 8, 10, 12]
  min_data_in_leaf: [20, 50, 100]
  feature_fraction: [0.6, 0.8, 1.0]
  bagging_fraction: [0.6, 0.8, 1.0]
  lambda_l1: [0.0, 0.01, 0.1]
  lambda_l2: [0.0, 0.01, 0.1]
```

**优化目标**：
- 主目标：最小化验证集 `MAPE` 或 `RMSE`。
- 约束：方向准确率 > 65%，避免只优化幅度不顾方向。

## 六、针对本项目的优先级建议

1. **先修复 `train_pipeline.py` 中小波分解与特征矩阵的对齐问题**。
2. **跑通纯 LGBM 原型，确定基线**（已完成，MAPE ~4.2%）。
3. **加入 WT-LGBM 对比**，看小波分解是否带来提升。
4. **用 Optuna 做一轮粗调**，锁定 `num_leaves`、`max_depth`、`learning_rate` 的大致范围。
5. **细调正则化和采样参数**，提升泛化。
6. **尝试稳健损失函数**（Huber/Quantile），看尖峰预测是否改善。

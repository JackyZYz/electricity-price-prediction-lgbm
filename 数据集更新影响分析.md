## 二、对数据接入层的影响

### 2.1 有利影响
- **Mock 数据可部分替换为真实数据**：原型设计中 `generate_mock_data()` 可由真实 CSV 读取替代，提升验证可信度。
- **DataReader 接口可用真实数据源验证**：原设计 `CSV/Excel 文件 · 数据库连接 · API 接口` 中的文件读取路径可直接对接 `Dataset/`。
- **Schema 字段可实际对齐**：原 `schema.py` 中 `TimeSeriesRecord`、`Granularity`、`DataSource` 等抽象定义可与真实列名、时间粒度对齐落地。

### 2.2 需要适配的变更
- **文件路径与命名约定需固化**：所有 CSV 均位于子目录下，文件命名规则为 `{类别}_{细分}_{发布类型}.csv`，需在 `data_sources.yaml` 中显式配置路径映射。
- **元数据列处理**：每个文件前 9 列为元数据列（ID、父ID、数据类型、数据地区、数据所属菜单、数据来源、数据描述、日期、更新时间），需统一识别并剥离。
- **时间列解析**：`日期` 列为日期维度，`00:00` ~ `24:00` 为 96 个时点列，需从宽表转长表（melt）。
- **`00:00` 列缺失约定**：多数文件 `00:00` 列为空、`24:00` 列有值，需在 `DataAdapter` 中增加“用 24:00 回填 00:00”或“统一舍弃 00:00”的清洗逻辑。

## 三、对特征工程的影响

### 3.1 可用特征清单
| 特征组 | 可用文件 | 建议用途 |
|--------|----------|----------|
| 需求侧 | 短期系统负荷预测（出清/边界）、实际系统负荷 | 直接特征、日前预测准确率偏差 |
| 供给侧 | 煤电发电计划、燃机固定出力总值、储能发电计划 | 火电/灵活资源供给曲线 |
| 新能源 | 统调光电功率预测、统调风电功率预测、实际统调风光情况 | 风光出力、新能源渗透率 |
| 联络线 | 受电计划、实际受电情况 | 外来电影响 |
| 安全裕度 | 正负备用空间 | 系统紧张程度 |
| 辅助 | 分省可再生能源发电能力预测 | 绿电供给能力参考 |
| 目标 | 统一结算点电价最终结果 | 主目标变量 |

### 3.2 不建议入模的特征
- **用户侧日前出清发布_出清电力.csv**：缺失率高达 47.87%，直接入模会引入大量缺失值，建议仅作为探索性分析或弃用。
- **统一结算点电价临时结果**：与最终结果高度一致，可作一致性校验，不建议与目标同时入模。

### 3.3 特征构造建议
- **净负荷** = 短期系统负荷预测 - 风光预测
- **新能源渗透率** =（风电预测 + 光电预测）/ 负荷预测
- **外来电占比** = 受电计划 / 负荷预测
- **火电占比** =（煤电计划 + 燃机出力）/ 负荷预测
- **日前/实际偏差** = 实际系统负荷 - 短期负荷预测
- **滞后特征**：电价 lag-1d、lag-7d、过去 24h/7d 滚动均价
- **时间特征**：hour、weekday、month、是否峰段（08-11、17-21）

## 四、对模型训练的影响

### 4.1 样本规模
- 目标变量有效样本约 17,568 条（183 天 × 96 时点）。
- 计划/预测类特征与电价日期交集约 176 天，有效样本约 16,896 条。
- 建议建模时以 176 天交集为基准，或仅使用前 176 天对齐数据。

### 4.2 训练策略调整
- **时序分割**：仍按原设计“最近 N 天为测试集”，但需确保训练/测试集按日期顺序切分，避免数据泄露。
- **缺失值处理**：LightGBM 原生支持缺失值，但 `00:00` 列与风光夜间空值需按业务规则预处理（如 0 填充、前后向填充）。
- **异常值处理**：电价最大值 1,251.20，远超均值 310.39，建议保留（现货市场真实尖峰）或采用分位数损失/Huber 损失。
- **目标变量选择**：统一使用“最终结果”，临时结果仅作校验。

### 4.3 原型方案影响
- 原 `prototype_lgbm.ipynb` 中的 Mock 数据生成器可保留作为 fallback，但主流程应优先读取 `Dataset/` 真实数据。
- 原型目标 MAPE < 15% 的验证可在真实数据上重新跑通，结果更具说服力。

## 五、对评估与验证的影响

### 5.1 评估指标
仍按原设计使用 MAE、RMSE、MAPE、R²、方向准确率、尖峰命中率、分时段评估。

### 5.2 新增验证项
- **日期对齐校验**：检查各文件日期集合与目标变量日期的交集。
- **缺失值监控**：重点监控 `出清电力` 及风光夜间空值。
- **数据漂移检测**：因数据为 2026 年半年数据，需关注季节漂移（1-7 月涵盖冬季到夏季）。
- **重复日期检查**：根目录 `分省可再生能源发电能力预测_发电能力.csv` 存在 3 个重复日期，需去重。

## 七、风险与注意事项

| 风险点 | 说明 | 建议措施 |
|--------|------|----------|
| 时间对齐不一致 | 计划/预测类文件比电价少 6 天 | 建模前取日期交集或补齐缺失日 |
| 高缺失率字段 | 出清电力缺失率 47.87% | 不入模，或仅作EDA |
| 00:00 列缺失 | 普遍存在于多数文件 | 统一用 24:00 回填或删除该列 |
| 风光夜间空值 | 物理意义为 0 | 按 0 填充 |
| 电价尖峰极端 | 最大值超过均值 4 倍 | 保留真实值，选用稳健损失函数 |
| 数据时效性 | 当前数据截至 2026-07-01 | 后续需设计每日增量更新机制 |
| 重复日期 | 可再生能源文件存在重复 | 去重后使用 |

## 一、业务背景

### 1.1 为什么要做这个

公司在江苏电力市场参与购电交易，每日需要在日前市场申报购电需求和报价策略。当前面临的问题：

- **购电成本不可控**：用户侧日前出清电价波动大，缺乏预判手段，购电决策依赖经验
- **信息利用率低**：交易中心披露了大量事前预测数据和事后结算数据，但没有系统性地用于辅助交易决策
- **风险暴露**：极端价格时段（如晚高峰、新能源低谷期）缺乏预警，高价买电侵蚀利润

### 1.2 业务场景

| 场景 | 说明 | 频率 |
|------|------|------|
| **日前购电报价** | 每天上午10点前，根据次日各时段预测电价制定购电策略 | 每日 |
| **中长期与现货分配** | 根据日前价格预测，决定多少电量走中长期、多少走现货 | 每周/每月 |
| **极端价格预警** | 预判次日是否出现尖峰电价，提前调整用电或启动需求响应 | 每日 |
| **结算复盘** | 对比预测价与实际出清价，评估预测准确率和交易策略效果 | 每周 |

## 二、项目目标

### 2.1 交付物

| 编号 | 交付物 | 说明 |
|------|--------|------|
| D1 | **电价预测模型** | 每天自动预测未来一天用户侧日前出清电价（96点或24点） |
| D2 | **预测结果输出** | 每日定时生成预测文件（CSV/Excel），或接入公司现有交易系统 |
| D3 | **模型评估报告** | 月度预测精度报告（RMSE、MAE、MAPE、方向准确率） |
| D4 | **特征重要性分析** | 输出对用户侧电价影响最大的TOP10因素，辅助交易员理解市场 |

### 2.2 精度目标

| 指标 | 目标值 | 备注 |
|------|--------|------|
| MAPE（日均） | < 10% | 日前预测常用基准线 |
| 方向准确率 | > 65% | 预测涨跌方向正确率，交易决策关键 |
| 尖峰捕捉率 | > 70% | 价格超过均值+2σ的时段被正确预警的比例 |
| 模型更新频率 | 每日自动 | 新数据入库后自动重训练或增量更新 |

### 2.3 非功能需求

- **运行环境**：公司内部服务器或云服务器（Linux/Windows），需支持GPU可选
- **运行时间**：数据处理+模型推理 < 30分钟
- **数据安全**：数据不出公司内网
- **可维护性**：代码模块化，关键参数可配置，支持人工干预

## 四、技术方案

### 4.1 推荐方案：WT-LGBM 动态分解框架

选型理由：
- 三篇文献中王博方案最成熟、代码实现难度最低、自带防数据泄露机制
- LightGBM 训练快、可解释性强（特征重要性直接输出），适合工程落地
- 小波变换计算效率高，支持每日滚动更新

### 4.2 技术路线

```
┌──────────────────────────────────────────────────────┐
│                    每日定时任务                        │
├──────────────────────────────────────────────────────┤
│                                                      │
│  ① 数据拉取                                          │
│     从数据库/文件系统读取最新事前+事后数据            │
│                                                      │
│  ② 数据预处理                                        │
│     - 异常值检测（IQR）                               │
│     - 缺失值填补（前向填充 / 线性插值）               │
│     - 特征构造（竞价空间、净负荷、外来电占比…）      │
│     - 时序对齐（确保事前/事后时间戳一致）             │
│                                                      │
│  ③ 滚动窗口 + 小波分解                               │
│     - 取过去 S 天的历史用户侧电价                     │
│     - DWT 分解为低频（趋势）+ 高频（波动）            │
│                                                      │
│  ④ 预测                                              │
│     - 低频分量 → LightGBM（输入：事前特征+滞后特征）  │
│     - 高频分量 → LightGBM（同上）                      │
│     - 用户侧电价 = 低频预测 + 高频预测                │
│                                                      │
│  ⑤ 输出                                              │
│     - 写入预测结果表/文件                             │
│     - 更新模型评估指标                                │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 4.4 模型参数

| 参数 | 参考值 | 说明 |
|------|--------|------|
| 滑动窗口长度 S | 30天 | 小波分解的历史窗口 |
| 小波基函数 | db4 | Daubechies 4，常用选择 |
| 分解层数 | 2-3层 | 1层=高频+低频，2层=高频1+高频2+低频 |
| LGBM learning_rate | 0.05-0.1 | 超参调优确定 |
| LGBM num_leaves | 31-127 | 超参调优确定 |
| LGBM max_depth | 10-15 | 防止过拟合 |
| 训练/测试分割 | 最近30天为测试集 | 时序分割，不打乱；当前数据约 176 天有效交集 |
| 数据粒度 | 15min | 每日 96 时点，与真实数据集一致 |

### 4.5 备选方案

如果 Phase 1 的 WT-LGBM 效果不理想，备选方向：

| 备选方案 | 适用场景 | 额外工作 |
|----------|----------|----------|
| 不做分解，直接 LGBM | 分解带来的提升不明显 | 更简单，去掉DWT模块 |
| 高频用 TimesNet | 多周期模式显著（日/周/月） | 需要 PyTorch，训练时间增加 |
| 加入 Attention LSTM | 需要更强的非线性拟合 | 需要 GPU，可解释性下降 |

## 五、评价指标

### 5.1 模型精度指标

| 指标 | 计算 | 用途 |
|------|------|------|
| MAE | mean(\|ŷ - y\|) | 平均预测偏差，元/MWh |
| RMSE | √mean((ŷ - y)²) | 对大误差惩罚更强 |
| MAPE | mean(\|ŷ - y\| / y) × 100% | 相对误差百分比 |
| R² | 1 - Σ(ŷ-y)²/Σ(ȳ-y)² | 拟合优度 |

### 5.2 业务决策指标

| 指标 | 计算 | 用途 |
|------|------|------|
| **方向准确率** | 预测涨跌与实际涨跌一致的比例 | ★ 交易决策最核心指标 |
| **尖峰命中率** | 真实尖峰中被预测到的比例 | 极端价格预警效果 |
| **尖峰误报率** | 预测尖峰中实际不是的比例 | 误报造成的策略损失 |
| **分时段 MAPE** | 峰/平/谷各段分别计算 | 不同时段精度差异分析 |

## 八、风险提示

| 风险 | 可能性 | 应对 |
|------|--------|------|
| 数据字段不全，关键特征缺失 | 低 | 已确认核心供需/安全类数据齐全，缺失省间部分字段可通过滞后特征弥补 |
| 计划/预测类特征比电价少 6 天 | 高 | 建模时取日期交集（176 天），或补齐缺失日 |
| 出清电力缺失率过高 | 高 | 该字段不入模，仅作探索性分析 |
| 00:00 列普遍存在空值 | 高 | 统一用 24:00 回填或删除该列 |
| 风光夜间空值 | 中 | 按物理意义 0 填充 |
| 用户侧电价与发电侧电价几乎一致 | 高 | 不影响项目本身价值，预测模型仍然有效 |
| 事前预测数据质量差（如负荷预测偏差大） | 中 | 同时使用事后实际值作为输入（仅历史值），事前特征做鲁棒性处理 |
| 零电价/负电价导致MAPE爆炸 | 低 | 当前数据最小值 200.00，暂不会出现；后续采用 sMAPE 或 MAAPE 替代 |
| 电价尖峰极端 | 中 | 保留真实值，采用稳健损失函数（Huber/分位数损失） |
| 节假日/特殊事件数据少，模型泛化差 | 中 | 手工标注特殊日期，单独评估 |
